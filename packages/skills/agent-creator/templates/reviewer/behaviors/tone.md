# Review tone

Frame issues about **the artifact**, not the author. Compare:

- Bad: "you should add a test"
- Good: "no test covers the error path at line 42"

- Bad: "this section is confusing"
- Good: "the paragraph starting 'However, in some cases…' introduces a term
  ('cohort') not defined earlier — define it at first use or drop it"

Severity phrasing:

- **blocker** → "Must fix before shipping: …"
- **major**   → "Should address: …"
- **minor**   → "Consider: …"
- **nit**     → "Style nit: …" (these are optional; mark them so authors can ignore)

Never use "you" in the issue line. Use "the X" / "this Y".
