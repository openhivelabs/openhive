# Templates

Pick the closest one and adapt. You can always mix — e.g. start from `lead`
and add `examples/` from `writer`.

## lead

Orchestrator. Plans and delegates. No execution.

Key files:
- `knowledge/delegation-patterns.md` — canonical brief shapes
- `behaviors/escalation.md` — when to stop and ask the user

Best for: team Lead nodes, entry point for chat triggers.

## researcher

Specialist that gathers info and cites sources.

Key files:
- `knowledge/source-quality.md` — how to rank sources
- `examples/comparison-brief.md` — canonical A vs B structure

Best for: R&D teams, competitive analysis, due diligence.

## reviewer

Inspects artifacts and returns structured verdicts.

Key files:
- `knowledge/review-checklists.md` — prose / code / data checklists
- `behaviors/tone.md` — how to phrase issues

Best for: QA nodes, editor roles, approval gates.

## writer

Produces documents from briefs.

Key files:
- `knowledge/house-style.md` — voice, numbers, formatting
- `examples/executive-memo.md` — one-page exec memo shape

Best for: reporting pipelines, content generation teams.

## Not covered (add your own)

- `translator` — converts between languages with style preservation
- `planner` — produces schedules / sprint plans from goals
- `coder` — writes or modifies code (probably wants tools like edit_file)
- `moderator` — applies content policies to inbound messages

Fork the closest template and rename.
