# pdf troubleshooting

## "spec-level op requested but no .spec.json"

This PDF was not generated by `build_doc.py`. Options:
1. **Use original source** — rebuild from docx or md
2. **Extract then edit** — create a best-effort spec with `extract_doc.py --in X.pdf --out X.spec.json`, hand-edit it, then regenerate with build_doc. Layout becomes simpler (tables/images lost)

## Watermark is not visible

`overlay_text` x/y are **points from the bottom-left of the page**. A4 is 595×842 pt. For near-center placement, use around `x=150, y=400`. If the stamp lands outside the page, it is invisible.

## Korean renders as boxes

ReportLab built-in fonts (Helvetica / Times-Roman / Courier) are Latin. Proper Korean PDFs need CID/TTF font embedding, which is currently out of scope. For Korean documents, generating docx first and exporting to PDF from Word/LibreOffice may give better quality.

(Future extension: reportlab `TTFont` can embed fonts such as Noto Sans KR. Planned support: theme override that maps `body_font` to an embedded font.)

## Page number is clipped

If `margin_bottom` is too small, page numbers can be covered by content. Do not reduce below the default 54pt.

## Extracted text is wrong

PDF keeps only rendered glyph placement, so **reading order is not preserved**. Multi-column layouts, tables, and viewer-specific text-layer quality can scramble order. Trust extraction for inspection only; prefer `.spec.json` for round-trips.

## Editing digitally signed PDFs

The signature is invalidated. Any `edit_doc.py` operation, page op or spec op, breaks the signature hash. Keep signed PDFs as records; edit from original source.

## Encrypted PDFs

This skill currently does not pass passwords to pypdf. Encrypted inputs fail. If needed, decrypt first with something like `qpdf --decrypt`, edit, then re-encrypt.
